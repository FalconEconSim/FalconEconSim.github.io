§ 2.1 – 2.4

Indifference Curves

An indifference curve (IC) is a set of all bundles $(X, Y)$ giving a consumer the same utility. Their shape encodes everything about preferences — without ever asking the consumer to assign a dollar value to happiness.

Key Idea
Well-behaved ICs are downward-sloping (both goods desirable), convex (preference for variety), and never cross (transitivity of preferences).
▸ Indifference Curve Explorer
Preference Type Cobb-Douglas
Utility Function U(X,Y) = X^α · Y^(1-α)
IC Shape Smooth & convex
MRS Diminishing
The consumer substitutes X for Y at a decreasing rate — convex ICs reflect the preference for variety. Moving to higher ICs (upper-right) means higher utility. 
import numpy as np

# Cobb-Douglas: U(X,Y) = X^alpha * Y^(1-alpha)
# For a given utility level k, solve for Y:
#   k = X^a * Y^b  =>  Y = (k / X^a)^(1/b)

alpha = 0.5
beta  = 1.0 - alpha
X     = np.linspace(0.1, 10, 400)

for k in [1, 2, 3, 4]:          # utility levels
    Y = (k / X**alpha) ** (1 / beta)
    # plot(X, Y)  — each curve is one indifference curve

# Perfect substitutes: U(X,Y) = a*X + b*Y
# Indifference curve: Y = (k - a*X) / b  — straight line

# Perfect complements: U(X,Y) = min(X/r, Y)
# Kink at X = r*Y — consumer always wants ratio r:1

# MRS (Cobb-Douglas) = MUx / MUy = (alpha * Y) / ((1-alpha) * X)
# MRS is DIMINISHING — decreases as X increases along an IC

Deeper Intuition: Indifference curves are level curves of the utility function — exactly like contour lines on a topographic map. The utility surface $U(X,Y)$ rises as you move outward; each IC is a horizontal slice of that surface at a fixed height.

▸ Utility Surface & Level Curves (3D) 
import numpy as np

# Build a grid of (X, Y) values
X, Y = np.meshgrid(
    np.linspace(0.1, 8, 60),
    np.linspace(0.1, 8, 60)
)

alpha = 0.5
Z = X**alpha * Y**(1 - alpha)   # utility surface

# Each horizontal slice Z = k is one indifference curve
# In 2D: Y = (k / X^alpha)^(1/(1-alpha))
# The "contour lines" of Z(X,Y) are exactly the ICs

# This is why economists say ICs are "level curves"
# — they are the set of points at a constant utility level
Common Mistake
Indifference curves never cross. If they did, a consumer would be simultaneously indifferent between bundle A & B and between B & C, but prefer A to C — a contradiction of transitivity.
§ 2.5

The Budget Constraint

The budget constraint $P_X \cdot X + P_Y \cdot Y = I$ is the boundary of what's affordable. Its slope $-P_X/P_Y$ is the opportunity cost of $X$ in terms of $Y$.

Budget Constraint — slope-intercept form
$$Y = \frac{I}{P_Y} - \frac{P_X}{P_Y}\, X$$
Y-intercept: \(\,I / P_Y\) X-intercept: \(\,I / P_X\) Slope: \(\,-P_X / P_Y\)
▸ Interactive Budget Line
Budget Equation 5X + 4Y = 100
X-intercept (max X) 20.0
Y-intercept (max Y) 25.0
Slope (−Pₓ/Pᵧ) −1.25
The slope tells you: for each extra unit of \(X\) purchased, you must give up 1.25 units of \(Y\). 
import numpy as np

I, Px, Py = 100, 5, 4

x_int = I / Px          # max X if all income spent on X
y_int = I / Py          # max Y if all income spent on Y
slope = -Px / Py        # opportunity cost of X in terms of Y

X = np.linspace(0, x_int, 400)
Y = (I - Px * X) / Py  # rearranged budget constraint

# Budget equation: Px*X + Py*Y = I
# Rearranged:      Y = I/Py - (Px/Py)*X
print(f"X-intercept: {x_int}")
print(f"Y-intercept: {y_int}")
print(f"Slope:       {slope:.3f}")
§ 2.6

Income & Price Changes

Income changes shift the budget line in parallel (slope unchanged). Price changes pivot it around the fixed intercept.

ChangeY-interceptX-interceptSlopeType
Income ↑Moves outMoves outFixedParallel shift out
Income ↓Moves inMoves inFixedParallel shift in
$P_X$ ↓FixedMoves outFlatterPivot out
$P_X$ ↑FixedMoves inSteeperPivot in
$P_Y$ ↓Moves outFixedSteeperPivot out
▸ Shift vs. Pivot Explorer
Change Income ↑ 30%
Original slope −1.25
New slope −1.25
Fixed intercept Neither (parallel)
A parallel shift means only income changed. Both intercepts move proportionally; the slope is set by prices alone. 
# Base parameters
I, Px, Py = 100, 5, 4

def budget_line(I, Px, Py):
    x_int = I / Px
    slope = -Px / Py
    y_int = I / Py
    return x_int, y_int, slope

# INCOME INCREASE → parallel shift
I_new = I * 1.3
# slope unchanged: -Px/Py stays the same
# both intercepts scale by 1.3

# PRICE INCREASE for X → pivot
Px_new = Px * 1.3
# Y-intercept = I/Py is FIXED (Py unchanged)
# X-intercept = I/Px_new shrinks inward
# Slope = -Px_new/Py becomes steeper

x1, y1, s1 = budget_line(I, Px, Py)
x2, y2, s2 = budget_line(I_new, Px, Py)
print(f"Original slope: {s1:.3f}")
print(f"After income ↑: slope = {s2:.3f} (unchanged)")
§ 2.7

The Consumer Optimum

The optimal bundle maximizes utility subject to the budget. Two conditions must hold simultaneously:

Two Conditions for an Interior Optimum
Condition 1 — Budget Exhausted
$$P_X X^* + P_Y Y^* = I$$
Every dollar is spent.
Condition 2 — Tangency
$$\frac{P_X}{P_Y} = \frac{MU_X}{MU_Y} = MRS$$
Market trade-off = personal trade-off.
▸ Consumer Optimum — Find the Best Bundle
Method Tangency condition
Optimal X* 10.00
Optimal Y* 12.50
Utility at optimum 11.18
Budget check ✓ 100.00 = 100
Tangency check ✓ MRS = 1.25
The red star marks where the budget line is tangent to the highest reachable indifference curve. 
# Cobb-Douglas: U(X,Y) = X^alpha * Y^(1-alpha)
I, Px, Py = 100, 5, 4
alpha = 0.5

# Tangency condition: MUx/MUy = Px/Py
# MUx = alpha * X^(alpha-1) * Y^(1-alpha)
# MUy = (1-alpha) * X^alpha * Y^(-alpha)
# MUx/MUy = (alpha * Y) / ((1-alpha) * X) = Px/Py
# => Y = (1-alpha)/alpha * (Px/Py) * X

beta = 1 - alpha
Y_from_tangency = (beta / alpha) * (Px / Py)  # Y = k*X

# Plug into budget constraint: Px*X + Py*(k*X) = I
k = Y_from_tangency
X_star = I / (Px + Py * k)
Y_star = k * X_star

# Equivalently (Cobb-Douglas shortcut):
X_star = alpha * I / Px
Y_star = beta  * I / Py

print(f"X* = {X_star:.4f}")
print(f"Y* = {Y_star:.4f}")
print(f"Budget: {Px*X_star + Py*Y_star:.2f} = {I}")
Common Mistake
The tangency condition applies only to smooth, convex ICs. For perfect complements, use the kink condition $X/a = Y/b$. For perfect substitutes, compare $MU_X/P_X$ vs $MU_Y/P_Y$ and buy only the better-value good.
§ 2.8

Bang for the Buck

The tangency condition $P_X/P_Y = MU_X/MU_Y$ can be rearranged to a deeply intuitive form:

Bang-for-the-Buck Condition (rearranged tangency)
$$\frac{MU_X}{P_X} = \frac{MU_Y}{P_Y}$$
Utility per dollar must be equalized across all goods at the optimum
Key Idea
If $MU_X/P_X > MU_Y/P_Y$, the consumer gets more utility per dollar from $X$. They should shift spending toward $X$ until the ratios equalize. This principle applies everywhere resources are scarce — marketing budgets, hospital allocations, government programs.
▸ Bang-for-the-Buck Reallocation
MU(X) / P(X) — bang for buck on X —
MU(Y) / P(Y) — bang for buck on Y —
Verdict —
Action —
Adjust spending on X to find the point where both ratios equalize — that is the optimum. 
# Cobb-Douglas: U = X^alpha * Y^(1-alpha)
# MUx = alpha * X^(alpha-1) * Y^(1-alpha)
# MUy = (1-alpha) * X^alpha * Y^(-alpha)

alpha, I, Px, Py = 0.5, 100, 5, 4
spend_X = 50   # dollars on X
spend_Y = I - spend_X

X = spend_X / Px
Y = spend_Y / Py

MUx = alpha * X**(alpha-1) * Y**(1-alpha)
MUy = (1-alpha) * X**alpha * Y**(-alpha)

bang_X = MUx / Px
bang_Y = MUy / Py

print(f"MUx/Px = {bang_X:.4f}")
print(f"MUy/Py = {bang_Y:.4f}")
if bang_X > bang_Y:
    print("→ Shift spending toward X")
elif bang_X < bang_Y:
    print("→ Shift spending toward Y")
else:
    print("→ At the optimum!")
§ 2.9

Cobb-Douglas Demand Functions

For $U(X,Y) = X^\alpha Y^\beta$, the optimal demand functions have an elegant fixed-share property: the consumer always spends a constant fraction of income on each good.

Cobb-Douglas Demand Functions
$$X^* = \frac{\alpha}{\alpha+\beta} \cdot \frac{I}{P_X} \qquad Y^* = \frac{\beta}{\alpha+\beta} \cdot \frac{I}{P_Y}$$
The spending share \(\alpha/(\alpha+\beta)\) on good \(X\) is constant — independent of income and prices
Application
When economists report that "households spend roughly 30% of income on housing regardless of income level," they're observing Cobb-Douglas behaviour. The spending share $\alpha/(\alpha+\beta)$ is constant — independent of prices or income.
▸ Cobb-Douglas Demand — Spending Shares
Income share on X — α/(α+β) 50.0%
Income share on Y — β/(α+β) 50.0%
Optimal X* 10.00
Optimal Y* 12.50
Spending on X $50.00
Spending on Y $50.00 
# General Cobb-Douglas: U(X,Y) = X^alpha * Y^beta
alpha, beta = 0.5, 0.5
I, Px, Py   = 100, 5, 4

# Demand functions (from tangency + budget)
X_star = (alpha / (alpha + beta)) * (I / Px)
Y_star = (beta  / (alpha + beta)) * (I / Py)

share_X = alpha / (alpha + beta)   # constant — independent of I, Px, Py!
share_Y = beta  / (alpha + beta)

print(f"X* = {X_star:.4f},  Spending = ${Px*X_star:.2f}")
print(f"Y* = {Y_star:.4f},  Spending = ${Py*Y_star:.2f}")
print(f"Income share on X: {share_X:.1%}")
print(f"Income share on Y: {share_Y:.1%}")
